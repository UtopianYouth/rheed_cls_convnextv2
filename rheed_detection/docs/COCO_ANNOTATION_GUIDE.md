# RHEED 目标检测 COCO 标注规范（条纹 / 斑点）

本文档用于指导标注人员对 RHEED 图像中的 **条纹（stripe）** 与 **斑点（spot）** 进行目标检测标注，并统一为 **COCO 格式**，确保后续 RT-DETRv2 实验可快速复现。

---

## 1. 标注目标与类别定义

- **stripe（类别 id = 1）**：条纹状 RHEED 结构。
- **spot（类别 id = 2）**：亮斑或斑点状 RHEED 结构。

> 说明：2D 目录中以条纹为主，3D 目录中以亮斑为主；若同一图像出现两类目标，需同时标注。

---

## 2. 标注规则（核心）

### 2.1 条纹（stripe）标注
- 对 **连续、亮度明显** 的条纹区域进行框选。
- 一条条纹 = 一个框。
- 若条纹断裂但属于同一结构，可用 **一个较大框** 覆盖。
- 避免将背景噪声或非目标结构纳入框内。

### 2.2 斑点（spot）标注
- 对 **清晰、高亮、可辨识** 的斑点进行框选。
- 若斑点密集，可使用 **一个较大框** 覆盖整个密集簇。
- 亮度过弱、边界不清的噪声点不做标注。

---

## 3. 边界框绘制要求

- 框应 **紧贴目标外接矩形**，不宜过大。
- 若目标局部遮挡，仅标注 **可见部分**。
- 不同标注人员需保持一致的框选尺度。

---

## 4. 不标注的情况

以下情况 **不做标注**：
- 噪声点、模糊亮点；
- 边界不可辨的弱目标；
- 非条纹 / 非斑点结构。

---

## 5. COCO 标注字段约定（关键）

COCO JSON 结构至少包含：
- `images`：图像信息（id, file_name, width, height）
- `annotations`：目标框信息（id, image_id, category_id, bbox, area, iscrowd）
- `categories`：类别信息

### 5.1 categories 示例
```json
"categories": [
  {"id": 1, "name": "stripe"},
  {"id": 2, "name": "spot"}
]
```

### 5.2 bbox 格式
- 使用 COCO 标准：`[x_min, y_min, width, height]`
- 坐标单位为 **像素**

---

## 6. 数据拆分建议

- **训练集 / 验证集 / 测试集 = 7:2:1**
- 保证每个子集均包含 `stripe` 与 `spot` 样本

---

## 7. 标注质量控制

- 每 50 张图像抽查 5 张。
- 如遇不确定目标，优先标注并在备注中说明。

---

## 8. 数据目录建议（可选）

```
rehed_det_coco/
  images/
    train/
    val/
    test/
  annotations/
    instances_train.json
    instances_val.json
    instances_test.json
```

---

## 9. 交付要求（给标注人员）

- 图像文件保持原始命名，不修改文件名。
- JSON 文件可由标注工具导出或汇总生成。
- 所有标注需符合本规范，避免类别 id 混用。

---

如有任何标注疑问，请统一反馈后再集中修订。
